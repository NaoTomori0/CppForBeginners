# Переменные в С++
В предыдущих программах мы, если честно, почти ничего не делали — просто красиво выводили строки на экран. Интерактива для пользователей тут не было, программа разговаривала сама с собой.

Настоящая магия начинается, когда мы начинаем работать с данными. И здесь на сцену выходит одно из базовых понятий программирования — переменная.

Ваш компьютер похож на гигантский склад с миллиардами коробок. Каждая подписана, а внутри лежат какие-нибудь значения.

В C++ переменная — это именованная область памяти, где программа хранит данные. Поскольку C++ является языком со строгой типизацией, вы обязаны указать тип данных для каждой переменной при её создании.

# Объявление
Для объявления переменной нужно сначала написать **тип** (`type`), затем **имя** (`id`) и, желательно, присвоить **начальное значение** (`=`):

``` cpp
type id;
type id2 = ...;
```

# Типы данных. Int
Давайте начнём знакомство с одним из самых популярных типов данных в C++ — `int` (от слова integer). Он предназначен для работы с целыми числами.

В переменную такого типа можно записать любое целое от −2147483648 до 2147483647 (-2<sup>31</sup> до 2<sup>31-1</sup>) (о том, как хранить числа, которые выходят за этот диапазон, мы узнаем немного позже). Но если вы попробуете добавить туда строчку "Привет", то получите совсем не тот результат, который хотели.


``` cpp
int num1;
int num2 = 5;
```

Пока мы не заплыли на настоящую глубину, давайте договоримся о нескольких правилах оформления программ. Некоторые из них незначимы для работы программы, но зато прямо влияют на психологическое состояние тех, кто будет читать наш код (вероятнее всего, это будем мы сами).

1. *Имя переменной должно начинаться с буквы латинского алфавита(a-z, A-Z) или подчеркивания (_), но не с цифры. Ещё в имени не может быть специальных символов — например, @ или ! . Пример: 
	- `7eleven` — плохо
	- `sevenEleven` — ок
2. Не используйте в именах ключевые слова самого языка: `cout`, `return`, `int`, а то компилятор запутается и сломается. Разумеется, вы пока знаете далеко не все эти слова, но можете ориентироваться на подсветку синтаксиса в IDE: если имя переменной вдруг поменяло цвет, это сигнал, что её стоит переименовать
3. Регистр имеет значение: `Cat`, `CAT` и `cat` — три разные переменные

Есть несколько «конвенций именования» — программистских соглашений по подбору имён для переменных. Вот некоторые из них:

- camelCase: `numberOfCats` — самый распространённый стиль для имён переменных и функций среди разработчиков на C++ (и среди любителей верблюдов)
- snake_case: `number_of_cats` — питонисты обожают
- UPPER_SNAKE: `NUMBER_OF_CATS` — применяется для глобальных констант. Повышенный тон намекает — не надо пытаться менять значение этой переменной!

---

Чтобы закрепить, как работать с переменными такого типа, напишем программу, которая считывает значение числа с клавиатуры и выводит на экран его куб.

```cpp
#include <iostream>

using namespace std;

int main() {
    int value = 0;
    cout << "Введите число: ";
    cin >> value;
    cout << value << "^3 = " << value * value * value << endl;
    return 0;
}
```

Что происходит в этой программе? Сначала мы создаём и **инициализируем** переменную `value`. Затем выводим на экран призыв ввести число, а затем начинается что-то новенькое.

В третьей строке значение, которое пользователь вводит с клавиатуры, вносится в переменную `value`. Если ввести вместо цифры букву или другой символ, переменная останется пустой.

Обратите внимание: `cin` зеркально противоположно `cout`. Операторы >> и << — это как трубы: через `cin` данные втекают в переменную, через `cout` — вытекают на экран.
